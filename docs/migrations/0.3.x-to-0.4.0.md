# Migrating from oh-my-space 0.3.x to 0.4.0

0.4.0 은 manifest 파일명을 `sources.yaml` → `oms.yaml` 로, 데이터 디렉터리를 `sources/` → `oms/` 로 rename 합니다. CLI 명령 인터페이스(`oms sync`, `oms worktree add` 등) 는 그대로지만 디스크 레이아웃이 바뀌므로 워크스페이스마다 한 번 수동 마이그레이션이 필요합니다. 구 이름이 감지되면 `oms` 의 모든 명령이 차단합니다.

기본 절차 (0.3.x 의 `oms sync` 가 자동으로 추가한 `.gitignore` 의 `sources/` entry 가 그대로라면 디렉터리 자체는 untracked 상태이므로 `mv` 만으로 충분합니다):

```bash
# 1. manifest rename (tracked 라 git mv 사용)
git mv sources.yaml oms.yaml

# 2. 데이터 디렉터리 rename (untracked 라 일반 mv)
mv sources oms

# 3. .gitignore 의 'sources/' entry 를 'oms/' 로 수정
sed -i.bak 's|^/\{0,1\}sources/$|oms/|' .gitignore && rm .gitignore.bak

# 4. 새 레이아웃 검증
oms doctor

# 5. 변경 commit
git add . && git commit -m "chore: migrate to oh-my-space 0.4.0 (oms.yaml + oms/)"
```

`oms` 의 `.bare/` + worktree 메타데이터는 디렉터리 이름에 의존하지 않습니다. `.git` placeholder 의 `gitdir: ./.bare` 는 상대 경로이고, git worktree 의 path/gitdir 매핑도 `mv` 만으로 자동으로 따라오므로 위 절차로 충분합니다.

`sources/` 를 일부러 tracked 상태로 관리하던 경우 (`.gitignore` 에서 제외했다면) 2번 단계만 `git mv sources oms` 로 바꿔 주세요.
