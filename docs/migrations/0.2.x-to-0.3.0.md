# Migrating from oh-my-space 0.2.x to 0.3.0

0.2.x stored each alias as a Git submodule under `sources/<alias>/`. 0.3.0 replaces that with a bare clone + worktrees layout: `sources/<alias>/.bare/` holds the bare clone and `sources/<alias>/<branch>/` is the working tree. `oms sync` refuses to run while `.gitmodules` still registers a `sources/<alias>` path. Convert your workspace manually:

```bash
# 1. For each alias in sources.yaml, deinit and remove the submodule.
git submodule deinit -f sources/<alias>
git rm -f sources/<alias>
rm -rf .git/modules/<alias>

# 2. Drop the now-empty .gitmodules if nothing else uses it.
[ -s .gitmodules ] || rm -f .gitmodules

# 3. Add sources/ to .gitignore (oms sync will also do this).
grep -qxF 'sources/' .gitignore 2>/dev/null || echo 'sources/' >> .gitignore

# 4. Re-sync with the new model.
oms sync --all

# 5. Commit the result.
git add . && git commit -m "chore: migrate to oh-my-space 0.3.0 (bare + worktree)"
```

> Starting with 0.4.0, the manifest/directory names change again to `oms.yaml` / `oms/`. After migrating up to 0.3.x with the procedure above, continue with the [0.3.x → 0.4.0 guide](./0.3.x-to-0.4.0.md).
