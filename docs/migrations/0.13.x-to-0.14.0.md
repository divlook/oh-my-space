# Migrating from oh-my-space 0.13.x to 0.14.0

0.13.x exposed branch management across three top-level surfaces: `oms switch`, `oms checkout`, and the `oms branch` group (which owned only `list` and `delete`). 0.14.0 consolidates the whole branch lifecycle under a single namespace: `switch` and `checkout` are now **subcommands of `oms branch`**.

- **`oms branch switch [alias] [branch]`** — LOCAL branch management (formerly `oms switch`). Behavior, arguments, and the `--from <ref>` option are unchanged.
- **`oms branch checkout [alias] [branch]`** — REMOTE branch tracking (formerly `oms checkout`). Behavior and arguments are unchanged.

The interactive `oms branch` action selector now offers all four lifecycle actions in order: `list`, `switch`, `checkout`, `delete`.

This is a command-surface change only. Submodules, `oms.yaml`, and the on-disk layout are untouched — no re-sync is required.

## Breaking change

The top-level `oms switch` and `oms checkout` commands are **removed with no deprecated aliases**. Calling them fails as an unknown command and exits non-zero. Update every invocation (including scripts and CI) to the new subcommand path.

| 0.13.x | 0.14.0 |
| --- | --- |
| `oms switch <alias> <branch>` | `oms branch switch <alias> <branch>` |
| `oms switch <alias> <branch> --from <ref>` | `oms branch switch <alias> <branch> --from <ref>` |
| `oms switch` (interactive) | `oms branch switch` (interactive) |
| `oms checkout <alias> <branch>` | `oms branch checkout <alias> <branch>` |
| `oms checkout` (interactive) | `oms branch checkout` (interactive) |

## What stays the same

- The behavior of switch and checkout is identical — only their parent command changed. Omitting the alias and/or branch still prompts interactively, and a non-interactive shell still fails fast instead of hanging.
- `oms branch list` and `oms branch delete` are unchanged.
- All other commands (`sync`, `status`, `commit`, `record`, `fetch`, `pull`, `push`, `unsync`, `doctor`, `init`, `agent`, `skills`, `update`) are unchanged.
