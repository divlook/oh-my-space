# Migrating from oh-my-space 0.7.x to 0.8.0

0.7.x had a single `oms checkout <alias> <branch>` that did three things: switch to an existing local branch, check out a remote branch as a tracking branch, and create a brand-new local branch when the branch existed nowhere. 0.8.0 splits that into two focused commands:

- **`oms switch [alias] [branch]`** — LOCAL branch management. Switches to an existing local branch, or creates a new one (`--from <ref>` sets the start point). No remote is consulted, and a new branch sets **no** upstream tracking.
- **`oms checkout [alias] [branch]`** — REMOTE branch exploration. Runs `git fetch origin --prune`, then checks out an existing `origin/*` branch as a local tracking branch (or switches to an existing local counterpart).

Both commands now accept an **omitted alias and/or branch** and prompt for them interactively (synced submodules, and local or `origin/*` branches respectively). On a non-interactive shell (CI, pipes) an omitted argument fails fast instead of hanging.

This is a command-surface change only. Submodules, `oms.yaml`, and the on-disk layout are untouched — no re-sync is required.

## Breaking change

`oms checkout <alias> <branch>` **no longer creates a brand-new local branch**. Creating local branches now belongs to `oms switch`.

| 0.7.x | 0.8.0 |
| --- | --- |
| `oms checkout api feature/new` (branch exists nowhere → created locally) | `oms switch api feature/new` |
| `oms checkout api feature/new --from main` | `oms switch api feature/new --from main` |
| `oms checkout api dev` (`dev` exists on `origin` → tracking branch) | `oms checkout api dev` (unchanged) |
| `oms checkout api main` (`main` already local → just switch) | `oms switch api main` (or `oms checkout api main`) |

If you call `oms checkout <alias> <branch>` for a branch that exists neither locally nor on `origin`, it now fails with a message pointing you at `oms switch`.

## What stays the same

- Checking out an existing remote branch with tracking: `oms checkout <alias> <branch>` works as before (now it always fetches first).
- `--from <ref>` for new branches: moved from `checkout` to `switch`.
- All other commands (`sync`, `status`, `fetch`, `pull`, `push`, `unsync`, `doctor`, `init`) are unchanged.
